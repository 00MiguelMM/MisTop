# Qué es Agile y cuál es su objetivo en el desarrollo de software
Agile es una forma de gestionar proyectos y desarrollar software basada en la iteración.

Su objetivo es dividir proyectos grandes en fases más pequeñas para fomentar la colaboración, la flexibilidad y la entrega continua. Esto pone énfasis en la entrega incremental, el trabajo en equipo, la planificación continua y el aprendizaje constante.

En el Manifiesto Agile de 2001 se definen cuatro valores fundamentales que sirven como base del movimiento Agile:

- Individuos e interacciones sobre procesos y herramientas.
- Software funcional sobre documentación exhaustiva.
- Colaboración con el cliente sobre la negociación de contratos.
- Responder al cambio sobre seguir un plan.

# Investiga qué es Scrum y sus conceptos principales (roles, sprints, backlog, reviews)
Scrum es una metodología ágil que organiza el trabajo en periodos cortos llamados sprints. Su objetivo es desarrollar el producto de forma incremental, entregando mejoras continuas en cada ciclo.

## Roles
- Product Owner: define qué se debe hacer y prioriza las tareas.
- Scrum Master: ayuda al equipo y elimina obstáculos.
- Equipo de desarrollo: se encarga de construir el producto.

## Sprints
Son periodos de trabajo cortos (normalmente de 1 a 4 semanas) en los que se desarrollan una serie de tareas concretas.

## Backlog
- Product Backlog: lista de todas las tareas o funcionalidades del proyecto.
- Sprint Backlog: conjunto de tareas seleccionadas para un sprint.

## Reviews
Al final de cada sprint se realiza una revisión (Sprint Review) para mostrar el trabajo realizado y recibir feedback.

# Investiga qué es Kanban y cómo se usa para organizar tareas
La metodología Kanban es un sistema visual de gestión de trabajo diseñado para optimizar el flujo, maximizar la eficiencia y limitar el trabajo en curso (WIP)
Aunque Kanban se creó para la fabricación, el desarrollo de software comparte muchos de los mismos objetivos, como aumentar el flujo y el rendimiento.

## Cómo se usa para organizar tareas
Se organizan las tareas a partir de los paneles kanban. Dichos paneles, utilizan tarjetas ordenadas según el progreso de cada proyecto. De esta forma tan visual, podemos saber en qué punto se encuentra cada proyecto en cada momento. 

# Explica las diferencias entre Scrum y Kanban
Tanto Scrum como Kanban son metodologías ágiles, que sirven para organizar proyectos.
Sin embargo, Scrum se utiliza para dividir el proyecto en partes más pequeñas, para que los desarrolladores trabajen de manera más flexible, puedan colabborar o trabajar en equipo.

Por otro lado, Kanban es una herramienta de organización más visual, que permite saber en cada momento cuál es el punto en el que se ecuentra cada proyecto.

# Describe en qué situaciones se podría usar cada metodología
Scrum se usa cuando quieres organizar el trabajo en sprints (bloques de tiempo)

Kanban se usa cuando el proyecto va cambiando o las tareas van surgiendo poco a poco. Es útil para verlas de forma visual y organizarlas fácilmente.
